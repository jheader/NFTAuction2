# NFTAuction2
基于Web3 生态搭建 NFT（ERC721）链上拍卖市场，实现数字艺术品的拍卖、出价、交割全流程上链
NFTAuction2 项目采用三层核心架构 + 第三方生态集成 + 数据存储层的混合架构设计，以以太坊区块链为底层核心，结合 Web2 开发技术实现链上链下协同，既保证 NFT 拍卖的去中心化和安全性，又兼顾平台的交互体验和数据处理效率，整体架构分层清晰、职责明确，以下是详细的架构拆解和技术细节：
一、核心三层架构（核心业务载体）
整体分为前端层、后端层、区块链节点层，三层各司其职且通过标准化接口交互，构成拍卖平台的核心运行体系。
1. 前端层（FrontEnd）：基于 JS 开发，用户交互入口
核心技术：采用 JavaScript 生态（Vue 等框架），集成Ethers等 Web3 SDK，搭配 MetaMask/WalletConnect 等加密钱包插件。
核心职责：作为用户与平台的交互窗口，实现钱包连接、拍卖操作、数据展示等功能；直接与区块链节点交互，不经过后端中转敏感交易。
关键交互逻辑：根据合约 ABI 构造交易数据→用户钱包本地签名（私钥全程留存本地）→将签名后的交易通过 RPC 接口发送至以太坊节点，完成链上状态修改。
核心能力：支持交易发起、链上事件监听、拍卖数据实时回显，实现 “从用户点击到链上执行” 的闭环。
2. 后端层（BackEnd）：基于 Go 开发，链上链下数据桥梁
核心技术：Go 语言开发核心服务，搭配 REST API 提供接口能力，支持 HTTP/WS 协议进行事件订阅。
核心职责：不参与链上交易签名，仅作为数据中转和处理中心，衔接前端与区块链节点，同时集成第三方平台丰富数据。
关键能力
提供前端所需的所有 API（拍卖列表查询、出价历史、平台统计、NFT 列表查询等）；
订阅 / 轮询区块链合约的日志事件（创建拍卖、出价、结束拍卖等），完成链上链下数据同步，且处理网络中断、服务器重启等异常，保证数据可靠性；
集成第三方 API（Alchemy/Etherscan/OpenSea 等），查询钱包 NFT、NFT 地板价等数据。
3. 区块链节点层（Ethereum Nodes）：智能合约为核心，链上业务执行层
核心载体：以太坊公链节点，部署ERC721 NFT 拍卖专属智能合约，采用UUPS 可升级模式，支持后续功能迭代。
核心技术：合约编译后生成JSON 格式 ABI 接口（标准化交互规范），节点对外提供HTTP-RPC 接口，支持多语言（Go/JS/Rust）调用。
核心职责：执行所有拍卖的链上逻辑，包括 NFT 锁定、资金托管、出价验证、拍卖交割等；触发合约事件并生成日志，供后端订阅同步。
核心合约逻辑：通过两个 mapping 映射管理全平台拍卖（nftToken2AuctionId关联 NFT 与拍卖 ID，auctionData存储拍卖详情），提供 createAuction/bidAuction/endAuction 等核心接口。
二、数据存储层：MySQL+Redis，高效管理链下数据
作为三层架构的辅助层，负责存储平台的非链上核心数据，兼顾数据持久化和高并发访问：
MySQL：用于持久化存储平台结构化数据，包括拍卖详情、用户出价历史、平台统计数据、NFT 基础信息等，支持复杂的查询、排序、过滤操作。
Redis：作为高性能缓存和实时数据存储，缓存链上高频查询数据（如 NFT 地板价、当前拍卖最高价）、用户会话信息，同时支持实时事件推送，降低区块链节点的查询压力，提升前端响应速度。
三、第三方生态集成：丰富平台能力，降低开发成本
集成行业主流 Web3 平台的 API 和服务，补足平台数据和节点能力，无需自研全量功能：
节点服务：Alchemy/Infura，替代自建以太坊节点，提供稳定的 RPC 接口，支持交易发送和链上数据查询；
NFT 数据：Alchemy/Etherscan/Moralis，快速查询任意钱包地址的 NFT 持有列表，无需遍历区块链全量数据；
NFT 市场数据：OpenSea/MagicEden，查询 NFT 集合的地板价，为用户提供市场参考；
可选拓展：预言机服务，用于获取 ETH / 稳定币（USDT/USDC/DAI）的 USD 实时价格，支持多代币出价的价格验证。
四、架构核心交互规则（跨层协作标准）
ABI 接口为通用语言：合约编译生成的 ABI 是前后端与合约交互的唯一标准，所有链上函数调用均需遵循 ABI 的参数和格式定义；
前端直连链上，保证去中心化：前端通过 SDK 直接与区块链节点交互，交易签名由用户钱包本地完成，后端不接触用户私钥，从底层保证资产安全；
后端仅做数据处理，不干预交易：后端仅订阅链上事件、同步数据、提供 API，不参与任何链上交易的构造和签名，避免中心化风险；
多语言适配链上调用：基于以太坊节点的 RPC 接口 + 合约 ABI，Go（后端）、JS（前端）、Rust 等任意语言均可实现链上合约函数调用，适配不同开发场景。
五、架构核心优势
去中心化 + 安全性：核心交易上链，NFT 和资金由智能合约托管，私钥本地留存，避免平台中心化托管的资产风险；
高性能 + 高体验：通过 Redis 缓存、后端数据预处理，降低区块链节点的查询压力，实现前端数据的实时展示，兼顾 Web3 的安全性和 Web2 的交互体验；
可扩展性 + 可升级：合约采用 UUPS 可升级模式，后端提供标准化 API，前端可灵活迭代功能，同时支持稳定币支付、平台代币空投等拓展需求；
低成本 + 高效开发：集成第三方平台的成熟 API 和节点服务，无需自研 NFT 数据查询、节点部署等功能，降低开发和运维成本。
